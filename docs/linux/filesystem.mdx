# 探索 Linux 文件系统：一切皆文件

在上一篇简介中，我们提到了 Linux 的一个核心哲学——“一切皆文件”。这个概念是理解 Linux 工作方式的基石。本篇教程将带你深入探索 Linux 的文件系统结构，了解那些神秘的目录都是做什么用的。

## 文件系统层次结构标准 (FHS)

与 Windows 不同，Linux 没有“C盘”、“D盘”这样的盘符概念。相反，所有的文件和目录都始于一个单一的“根”目录，用斜杠 `/` 表示。这个结构被称为**文件系统层次结构标准（Filesystem Hierarchy Standard, FHS）**。FHS 定义了 Linux 系统中主要目录的用途，确保了不同 Linux 发行版之间的一致性。

想象一下，整个文件系统就像一棵倒置的树，` / ` 就是树根，所有其他目录都是从这个根上长出的分支。

![Linux 文件系统树](/img/docusaurus/linux-filesystem-tree.png)

下面，我们来认识一下这棵“树”上一些最重要的目录。

## 核心目录详解

| 目录 | 完整名称 | 描述 |
| :--- | :--- | :--- |
| **`/`** | Root | **根目录**。所有文件和目录的起点，万物之源。 |
| **`/bin`** | Binaries | 存放所有用户（包括普通用户和管理员）都可以执行的**基本命令**，如 `ls`, `cp`, `cat`。这些是系统启动和运行所必需的。 |
| **`/sbin`** | System Binaries | 存放只有**系统管理员（root）**才能执行的系统管理命令，如 `fdisk`, `ifconfig`, `reboot`。 |
| **`/etc`** | Etcetera | 存放整个系统的**配置文件**。几乎所有服务的配置都在这里，例如网络设置、用户密码、启动脚本等。非常重要！ |
| **`/home`** | Home | **用户的主目录**。每个普通用户在这里都有一个以其用户名命名的子目录，用于存放个人文件、文档和配置。例如，用户 `tamochii` 的主目录就是 `/home/tamochii`。 |
| **`/root`** | Root's Home | **超级用户（root）** 的主目录。与 `/home` 分开，以确保系统关键用户的环境独立和安全。 |
| **`/var`** | Variable | 存放**经常变化的文件**。例如日志文件 (`/var/log`)、邮件队列 (`/var/spool/mail`)、Web 服务器内容 (`/var/www`) 等。 |
| **`/tmp`** | Temporary | 存放**临时文件**。系统和应用程序会在这里创建临时文件，通常系统重启后这里的内容会被清空。 |
| **`/usr`** | Unix System Resources | **用户安装的软件和资源**。这是系统中最大的目录之一，包含了用户安装的应用程序、库文件、文档等。可以把它看作是 Windows 下的 `C:\Program Files`。 |
| | `/usr/bin` | 用户安装的应用程序的可执行文件。 |
| | `/usr/sbin` | 用户安装的系统管理程序。 |
| | `/usr/lib` | 应用程序使用的库文件。 |
| | `/usr/share/doc` | 软件包的文档。 |
| **`/boot`** | Boot | 存放**系统启动所需的文件**，包括 Linux 内核、引导加载程序（如 GRUB）的配置文件。不要轻易动这里的文件！ |
| **`/dev`** | Devices | **设备文件**。这是“一切皆文件”思想的完美体现。在 Linux 中，硬件设备（如硬盘、鼠标、摄像头）都以文件的形式出现在这里。例如，`/dev/sda` 可能代表你的第一块硬盘。 |
| **`/proc`** | Processes | 一个**虚拟文件系统**，它并不存在于磁盘上，而是由内核在内存中动态创建。它包含了系统和进程的实时信息。例如，你可以通过读取 `/proc/cpuinfo` 文件来查看 CPU 信息。 |
| **`/sys`** | System | 类似于 `/proc`，也是一个虚拟文件系统，提供了对内核数据结构和硬件属性的更现代、更结构化的访问方式。 |
| **`/lib`** | Libraries | 存放系统和应用程序运行所必需的**共享库文件**。这些库文件为 `/bin` 和 `/sbin` 中的命令提供支持。 |
| **`/opt`** | Optional | 用于存放**可选的、第三方软件包**。有些软件（如 Google Chrome）会选择安装在这里，而不是 `/usr`。 |
| **`/mnt`** | Mount | 临时**挂载点**。用于临时手动挂载外部设备，如 U 盘、移动硬盘等。 |
| **`/media`** | Media | 现代 Linux 系统中，用于**自动挂载可移动媒体设备**（如 U 盘、光盘）的目录。当你插入一个 U 盘时，系统通常会自动在 `/media` 下创建一个目录并挂载它。 |

## 绝对路径 vs. 相对路径

在 Linux 中定位文件或目录时，我们使用路径。路径分为两种：

- **绝对路径 (Absolute Path)**：
  - 从根目录 `/` 开始的路径。
  - 它唯一地标识了文件系统中的一个位置，无论你当前在哪个目录下。
  - 示例：`/home/tamochii/documents/report.txt`

- **相对路径 (Relative Path)**：
  - 从**当前工作目录**开始的路径。
  - 它依赖于你目前所在的位置。
  - 示例：假设你当前在 `/home/tamochii` 目录下，那么 `documents/report.txt` 就是一个相对路径。

特殊符号：
- `.` (一个点): 代表**当前目录**。
- `..` (两个点): 代表**上一级目录**（父目录）。

## 结语

理解 Linux 的文件系统结构是至关重要的第一步。它就像是学习一座城市的地图，知道了主要街道和地标的位置，你才能自如地穿梭其中。
